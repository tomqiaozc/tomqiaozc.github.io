01

Mostly handwritten code

Direct implementation
VS CodeHandwriting + autocomplete

Before AI coding tools became common, I implemented and verified code directly in the IDE.

02

Use AI to generate code snippets

Direct implementation
ChatGPTCopilot completionVS Code

I first generated function snippets in chat and copied them into the project, then used editor completion for simple tasks. AI supplied fragments; I integrated, revised, and verified them.

Limitation: repeatedly copying between chat, the editor, and the repository added friction.

03

Using Coding Agent CLI

Organization Execution · Serial
OpenCodeCoding CLI

The terminal became the main interface. A Coding Agent could read the repository, edit files, and run checks, so I focused more on specifying tasks and validating results.

04

Let a single Agent undertake the complete task

Organization Execution · Serial
Claude Code

A single Agent could complete more implementation and verification steps, but the workflow remained serial: assign a task, wait, then review it manually.

Next: separate independent tasks and run them in parallel.

05

Execute multiple tasks in parallel

Organize Execution · Parallel
Isolate workspaceParallel Agent

I split work into independently verifiable tasks and ran them in isolated workspaces. My focus shifted to dependency management, conflict control, and final integration.

06

Increase memory, independent review and quality checks

Organization Execution → Design Workflow
retrievable memoryIndependent Agent ReviewQuality check

Parallel execution exposed two problems: new Agents lacked prior context and repeated mistakes, while an Agent reviewing its own work could miss issues. I therefore stored retrievable lessons and assigned reviews to an independent Agent.

Next: encode these rules in one execution workflow.

07

Organize multiple agents with unified workflow

Design workflows
Agent TeamHarness execution loopExecution record

This workflow, also called a Harness, constrains how multiple Agents execute development tasks. Implementation, testing, and independent review move through explicit states with recorded evidence. A failed check returns the task to the relevant step; only accepted work proceeds to delivery.

I no longer implement every change myself, but I still need to understand the code, review critical changes, and perform final acceptance. My main responsibilities are now todecompose tasks, define execution rules, arrange independent Agent reviews, and remain accountable for the final result..$ Work focus: direct implementation → organization and execution → design workflow and review
