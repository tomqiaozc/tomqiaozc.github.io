01

主要依靠手写代码

直接实现
VS Code手写 + 自动补全

在 AI Coding 工具普及前，我在 IDE 中直接编写和验证代码，开发任务主要由人完成。

02

用 AI 生成代码片段

直接实现
ChatGPTCopilot 补全VS Code

先在对话中生成函数片段并复制回项目，后来使用编辑器补全处理简单需求。AI 提供片段，我负责整合、修改和验证。

问题：在对话、编辑器和项目之间反复复制，操作成本较高

03

使用 Coding Agent CLI

组织执行 · 串行
OpenCodeCoding CLI

工作入口转向终端。Coding Agent 可以读取仓库、修改文件并运行检查，我开始把重点放在描述任务和验收结果上。

04

让单个 Agent 承接完整任务

组织执行 · 串行
Claude Code

Agent 可以完成更多实现和验证步骤，但流程仍是“下达任务 → 等待完成 → 人工检查”，同一时间通常只推进一个任务。

下一步：把独立任务拆开并行执行

05

并行执行多个任务

组织执行 · 并行
隔离工作区并行 Agent

把工作拆成可独立验收的任务，在隔离工作区中并行推进。我的重点转向依赖管理、冲突控制和统一验收。

06

增加记忆、独立评审和质量检查

组织执行 → 设计工作流
可检索记忆独立 Agent 评审质量检查

并行执行后出现两个问题：新 Agent 不知道过去的经验，容易重复错误；同一 Agent 既实现又评审，容易漏掉问题。因此我保存可检索的经验，并安排未参与实现的 Agent 评审。

下一步：把这些规则放进统一的执行流程

07

用统一工作流组织多 Agent

设计工作流
Agent TeamHarness 执行循环执行记录

这套工作流也称为 Harness，用来约束多个 Agent 如何执行开发任务。实现、测试和独立评审按明确状态推进，过程有记录；检查失败时可以回退或重新执行，通过验收后再进入交付。

现在，我不再亲手完成所有实现，但仍需要理解代码、复核关键改动并完成最终验收。我的主要工作变为拆分任务、定义执行规则、安排独立 Agent 评审，并对最终结果负责。$ 工作重心：直接实现 → 组织执行 → 设计工作流与评审
